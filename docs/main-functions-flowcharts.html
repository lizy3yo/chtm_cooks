Company-standard process flow for the borrow request lifecycle, aligned to role handoff, status control, and operational accountability.

Actors: Student, Instructor, Custodian
Pattern: Controlled state transition workflow
Scope: Happy path with operational checkpoint

A. End-to-End Lifecycle Flowchart

flowchart TD
  classDef term fill:#DBEAFE,stroke:#1E3A8A,color:#1E3A8A,stroke-width:1.4px;
  classDef process fill:#F8FAFC,stroke:#475569,color:#0F172A,stroke-width:1.2px;
  classDef decision fill:#F3E8FF,stroke:#7E22CE,color:#581C87,stroke-width:1.3px;
  classDef io fill:#ECFEFF,stroke:#0E7490,color:#164E63,stroke-width:1.2px;
  classDef state fill:#FFF7ED,stroke:#C2410C,color:#7C2D12,stroke-width:1.3px;
  classDef done fill:#F0FDF4,stroke:#16A34A,color:#14532D,stroke-width:1.3px;
  classDef risk fill:#FEF2F2,stroke:#DC2626,color:#7F1D1D,stroke-width:1.2px;

  START([Start]):::term --> SUBMIT[/Student submits borrow request/]:::io
  SUBMIT --> ST1[[pending_instructor]]:::state
  ST1 --> REVIEW[Instructor reviews request]:::process
  REVIEW --> DECIDE{Instructor approves?}:::decision
  DECIDE -->|Yes| ST2[[approved_instructor]]:::state
  DECIDE -->|No| REJECT[[rejected]]:::risk
  REJECT --> END_REJ([End - Request Closed]):::term

  ST2 --> PREPARE[Custodian prepares items]:::process
  PREPARE --> RELEASE[Custodian releases request]:::process
  RELEASE --> ST3[[ready_for_pickup]]:::state
  ST3 --> PICKUP[/Student picks up items/]:::io
  PICKUP --> ST4[[borrowed]]:::state
  ST4 --> ACTIVE[Active borrow period]:::process
  ACTIVE --> RETURN[/Student returns items/]:::io
  RETURN --> ST5[[returned]]:::done
  ST5 --> END_OK([End - Completed]):::term

B. Role-Based Handoff Flow (Swimlane Style)

flowchart TB
  classDef lane fill:#F8FAFC,stroke:#CBD5E1,color:#0F172A,stroke-width:1.1px;
  classDef step fill:#FFFFFF,stroke:#94A3B8,color:#1E293B,stroke-width:1.1px;
  classDef io fill:#ECFEFF,stroke:#0E7490,color:#164E63,stroke-width:1.2px;
  classDef decision fill:#F3E8FF,stroke:#7E22CE,color:#581C87,stroke-width:1.2px;
  classDef status fill:#FEF3C7,stroke:#D97706,color:#78350F,stroke-width:1.2px;
  classDef finish fill:#DCFCE7,stroke:#16A34A,color:#14532D,stroke-width:1.2px;

  subgraph L1[Student Lane]
    ST1[/Create borrow request/]:::io
    ST2[/Pick up approved request/]:::io
    ST3[Use borrowed items]:::step
    ST4[/Return items/]:::io
  end
  class L1 lane

  subgraph L2[Instructor Lane]
    IN1[Review request details]:::step
    IN2{Approve request?}:::decision
  end
  class L2 lane

  subgraph L3[Custodian Lane]
    CU1[Prepare requested items]:::step
    CU2[Release request for pickup]:::step
  end
  class L3 lane

  ST1 --> S01[pending_instructor]:::status
  S01 --> IN1
  IN1 --> IN2
  IN2 -->|Yes| S02[approved_instructor]:::status
  IN2 -->|No| S99[rejected]:::status
  S02 --> CU1
  CU1 --> CU2
  CU2 --> S03[ready_for_pickup]:::status
  S03 --> ST2
  ST2 --> S04[borrowed]:::status
  S04 --> ST3
  ST3 --> ST4
  ST4 --> S05[returned]:::status
  S05 --> DONE([Completed]):::finish
pending_instructor: Request submitted by Student and awaiting Instructor decision.
approved_instructor: Instructor validated request; Custodian fulfillment starts.
ready_for_pickup: Custodian has released request; Student may claim items.
borrowed: Items are in active use during borrow period.
returned: Process completed and transaction closed.
Shape Standard: Terminator = rounded node, Process = rectangle, Decision = diamond, Input/Output = parallelogram, Status = highlighted state block.
