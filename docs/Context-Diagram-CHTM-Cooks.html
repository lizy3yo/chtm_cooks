System Overview

  • CHTM Cooks Inventory Management System: A comprehensive web-based platform for managing culinary equipment, tools, and supplies for the College of Hospitality and Tourism Management
  • Purpose: Streamline inventory tracking, request processing, donation management, and reporting for educational culinary operations
  • Architecture: Built with SvelteKit framework, utilizing server-side rendering and API routes for secure data management
  • Database: Centralized data storage for users, inventory items, requests, donations, inspections, and reports

External Entities

  • Superadmin: System administrator with full access to user management, system configuration, and comprehensive reporting capabilities
  • Instructor: Faculty member who monitors inventory usage, views student data, accesses analytics, and oversees donation records
  • Custodian: Operational staff responsible for inventory management, request processing, donation recording, and item inspections
  • Student: End-user who browses the catalog, submits item requests, tracks request status, and reports item issues

Key System Functions

  • Authentication & Authorization: Role-based access control ensuring appropriate permissions for each user type
  • Inventory Management: Complete CRUD operations for items including categorization, stock tracking, and archival
  • Request Processing: Workflow management for student requests with approval/rejection capabilities
  • Donation Management: Recording and integration of donated items into the inventory system
  • Item Inspection: Quality control and condition tracking for all inventory items
  • Reporting & Analytics: Comprehensive reporting tools for inventory status, usage patterns, and operational insights

Data Stores

  • Database Storage: Primary relational database containing all system data with proper normalization and indexing
  • Inventory Storage: Specialized storage for inventory item details, categories, quantities, and status information
  • Request Storage: Repository for all student requests, approval workflows, and historical request data
